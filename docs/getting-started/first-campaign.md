# Create your first campaign

A campaign is a persistent story instance created from one exact published world version.

## Create from the dashboard

1. Return to **Dashboard**.
2. Search or browse the available-worlds carousel and select the intended world card.
3. Review **World Details**, then select **Create campaign**.
4. Enter a campaign name.
5. Choose the player character.
6. Select **Create campaign**.

The basic dashboard form applies the current system defaults for settings such as provider, story response length, and turn control. Select **Advanced creation** to open **Setup → Campaign Management** when those values need to be changed before creation.

Nexus copies the selected character and exact published world version into the campaign. Later changes to the world roster or a newly published version do not silently change that campaign snapshot.

## Review campaign settings

Open **Setup → Campaign Management** and select the new campaign when you need to review or change:

- **Campaign title**
- **Status**
- **World version**
- **Text provider profile**
- **Default story response length**
- **Turn control style**

Confirm that the text provider is enabled and has a model. The story response length is a campaign preference; the provider's maximum output remains a separate safety ceiling.

Select **Save campaign** after changing settings.

## Start Infinite Quest

Completing the basic dashboard form immediately opens the campaign-specific Infinite Quest route and starts its first turn. Nexus hands off the server-generated accepted history without placing provider credentials in browser storage.

For a new campaign, Infinite Quest shows the background story when available and automatically submits the world's opening action. Configure the provider before creating the campaign so this first generation can start successfully.
