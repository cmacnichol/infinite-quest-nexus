# Project history and planning records

These documents record completed implementation checkpoints or explicitly deferred work. They are useful for understanding repository history, but they are not authoritative operating instructions.

## Completed checkpoints

- [Phase 1 checkpoint](../operations/phase-1-checkpoint.md): PostgreSQL and Chronicle baseline
- [Phase 2 checkpoint](../operations/phase-2-checkpoint.md): World Library and campaign persistence
- [Phase 3 checkpoint](../operations/phase-3-checkpoint.md): isolation, illustrations, and deployment work

Durable behavior from these checkpoints is incorporated into active guides and concept pages. If a checkpoint conflicts with current code or active documentation, the current implementation and active guide take precedence.

## Deferred work

- [Deferred improvements](../operations/deferred-improvements.md)

Deferred improvements describe planned or investigated behavior. They must not be read as implemented features. New work should normally be tracked in GitHub issues and recorded in an ADR when it changes a consequential architecture decision.
